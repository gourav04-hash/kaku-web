## Overview

**Kaku** is a full-stack telehealth and pharmacy management platform built with Next.js 16 (App Router). It serves four distinct user roles — **Patient**, **Doctor**, **Pharmacist**, and **Admin** — each with dedicated route groups, dashboards, and server actions. The system manages the full clinical lifecycle: appointment booking, video consultation, EMR (electronic medical records), prescription issuance, lab reports, pharmacy order fulfillment, and patient consent/privacy controls.

```
┌─────────────────────────────────────────────────────────┐
│                 Browser / Client (React 19)             │
│  /login  /register  /dashboard  /patient  /doctor      │
│  /pharmacist  /admin  /(telehealth)/[role]/...         │
├─────────────────────────────────────────────────────────┤
│                 Next.js App Router (SSR)                │
│   Server Actions (/src/lib/actions/)                    │
│   Route Handlers (/src/app/api/)                        │
│   Middleware (JWT RBAC enforcement)                     │
├─────────────────────────────────────────────────────────┤
│                 Data / Auth Layer                       │
│   Prisma ORM  ←→  PostgreSQL (Neon / Vercel Postgres)  │
│   NextAuth.js (JWT sessions)                           │
│   bcryptjs (password hashing)                          │
└─────────────────────────────────────────────────────────┘
```

## Data Flow

### 1. Authentication
```
POST /login → NextAuth CredentialsProvider → bcrypt.compare → JWT issued
JWT → middleware (Edge) → role check → authorized or redirect
```

### 2. Appointment Booking
```
Patient UI → bookAppointment() Server Action → Prisma INSERT Appointment
→ Notification created for Doctor → Doctor joins via /dashboard
```

### 3. Video Consultation
```
/(telehealth)/[role]/appointments/[id]/room
→ ConsultationRoom component → roomId from Appointment.roomId
→ External VIDEO_SDK (optional, env var VIDEO_SDK_API_KEY)
```

### 4. Prescription → Order Flow
```
Doctor issues Prescription (Server Action) → PrescriptionItems stored
→ Patient views Prescription → Checkout Modal → Order.create()
→ Pharmacist dashboard pulls pending Orders → fulfills → status DELIVERED
→ PaymentRecord created
```

### 5. EMR & Consent
```
Patient grants Consent to Doctor → ConsentToggle component
→ Doctor accesses /doctor/patients/[id]/history
→ EMR entries, diagnoses, documents read/written
→ AuditLog entry created on every record access
```

### 6. File Upload
```
POST /api/upload → multipart → saved to /public/uploads (or AWS S3 if configured)
→ MedicalDocument.url stored in Prisma
```

## Conventions

**Naming:** PascalCase for components, camelCase for actions, kebab-case for files  
**Structure:** Feature-first components in `src/components/features/`, shared in `src/components/shared/`, primitives in `src/components/ui/`  
**Data access:** Server Actions only — no API routes for data mutations (except `/api/auth` and `/api/upload`)  
**Auth pattern:** `getServerSession(authOptions)` called directly in server actions for identity  
**Forms:** `react-hook-form` + `zod` validation  
**Testing:** Not present — no test framework configured
